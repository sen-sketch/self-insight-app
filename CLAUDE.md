# Shared Collaboration Policy (Baseline)

## このプロジェクトでの役割
- 人間：仕様・設計・判断を決める／実際のファイル編集を行う
- AI：原因整理と修正方法の提示のみを行う

## AIの基本動作方針（最重要）
- 原則：修正方法のみを具体的に貼り付けできる形で出力する
- どのファイルのどの位置に何を追記し、何を削除・置換するかを具体化する
- AIが勝手にファイルを修正・生成した体で書かない
- 実際の編集・保存・コミットは人間が行う

## AIにやってほしいこと（OK）
- エラー原因調査
- 最小修正案の提示
- 既存構造に合わせた修正指示
- import 追跡、型・変数の定義元特定
- 軽微なレイアウト調整方針の提示
- 日本語で回答

## AIにやってほしくないこと（NG）
- 仕様変更
- 状態意味の変更
- 新設計レイヤーの勝手な導入
- `any` / `as any` による型安全性破壊
- 依存関係の勝手な追加
- 全面書き換え前提の提案

## ファイル修正ルール
- 人間が明示した対象ファイルを優先する
- 他ファイル修正が必要な場合は実編集せず、必要ファイル一覧と要点だけ示して許可待ちする

## 修正提案の出力形式
- 原因調査：候補は最大3つ、確度順
- 修正案：対象ファイル名、手順ベース、変更影響を1行で示す

## 禁止される近道
- if の乱造
- 型エラー握りつぶし
- 無関係な箇所を巻き込む修正
- MVP範囲を超える最適化

# Project Variables

## プロジェクト概要
- プロジェクト名: 自己観察Webアプリ（仮）
- 目的:
  - 日々の思考・感情・行動を記録する
  - 1日/7日/30日の振り返りを集計する
  - LLMへ貼り付け可能なテキストを生成する
- 対象ユーザー:
  - 自分自身のみ（単一ユーザー前提）
- 対象プラットフォーム:
  - スマホ優先のWebアプリ
- 標準タイムゾーン:
  - `Asia/Tokyo`

## 使用技術
- フロントエンド: `Next.js 16 (App Router)`
- 言語: `TypeScript`
- スタイリング: `Tailwind CSS`
- データ保存:
  - MVP方針: ブラウザローカル保存
  - 実装方式: `storage` 層経由で `localStorage`（主要データ）+ `IndexedDB`（将来の添付・大容量拡張用）
  - DB設計資料上の想定: PostgreSQL
- バックエンド:
  - MVPでは外部サーバ送信なし
  - 将来導入時: `Next.js Route Handlers` + `PostgreSQL` + 認証（詳細は導入時に追記）

## 主なドキュメント
- 要件定義: `$HOME/self-insight-app/docs/01_requirements.md`
- 画面設計: `$HOME/self-insight-app/docs/02_screen_design.md`
- DB設計: `$HOME/self-insight-app/docs/03_db_design.md`

## 想定ディレクトリと責務
- `self-insight-app/`（実装リポジトリのルート想定）
- `app/`
  - 画面実装
- `components/`
  - UI部品
- `lib/`
  - 集計処理、エクスポート整形、日付処理
- `storage/`
  - ローカル保存、将来のDB接続切替点
- `docs/`（または本ドキュメント群リポジトリ）
  - 要件・画面・DB設計の管理

## MVPスコープ
- 実装対象:
  - タイムライン投稿
  - 習慣タスク管理と開始記録
  - 習慣開始時刻の週次表示
  - 運を上げる記録
  - メタ認知日記（1日1件）
  - 1日/7日/30日エクスポート
- 非対象:
  - API連携
  - 通知
  - 画像添付
  - マルチユーザー対応
  - 同期最適化

## 画面一覧
- S01 ダッシュボード
- S02 タイムライン
- S03 習慣トラッカー
- S04 運を上げる記録
- S05 メタ認知日記
- S06 エクスポート

## スコープ制約
- 既存構造を優先し、MVP範囲を超える提案をしない
- 型安全性を壊す近道を使わない
- 無関係な設計変更を持ち込まない
