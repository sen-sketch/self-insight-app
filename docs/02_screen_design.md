# 画面設計（MVP）

## 1. 画面一覧
- S01: ダッシュボード
- S02: タイムライン（自分用Twitter）
- S03: 習慣トラッカー
- S04: 運を上げる記録
- S05: メタ認知日記
- S06: エクスポート

## 2. 画面遷移
- S01 ダッシュボードから S02-S06 へ遷移
- S02-S06 は下部タブで相互遷移
- S05 メタ認知日記は当日日記があれば編集モードで表示

## 3. 画面詳細

## S01 ダッシュボード
### 目的
- 今日の入力状況と直近トレンドを一目で確認する。

### 主要UI
- 今日の記録サマリカード
  - 投稿数
  - 習慣開始数
  - 運記録数
  - メタ認知日記入力済み/未入力
- 今週の習慣開始時刻ミニグラフ
- 直近投稿3件
- クイック入力ボタン

### 主操作
- 各機能画面へ移動

## S02 タイムライン
### 目的
- 思考・感情を短文で時系列記録する。

### 主要UI
- 投稿フォーム
  - 本文（必須、最大280文字）
  - 気分スコア（1-5）
  - タグ（任意、カンマ区切り）
  - 投稿ボタン
- 投稿一覧
  - 日時
  - 本文
  - 気分
  - タグ
- フィルタ
  - 日付範囲
  - タグ

### バリデーション
- 本文未入力は保存不可

## S03 習慣トラッカー
### 目的
- 習慣の開始行動を可視化し、開始時刻の改善を追う。

### 主要UI
- タスク一覧（有効/無効）
- タスク作成モーダル
  - タスク名
  - 目標開始時刻（任意）
- 開始記録ボタン（各タスク）
- 週次グラフ
  - 日別開始時刻プロット
  - 平均開始時刻
  - 前週比（早まった/遅れた）

### 主操作
- タスクの追加/編集/無効化
- 開始記録の追加

## S04 運を上げる記録
### 目的
- 小さな挑戦の蓄積と学びの再利用。

### 主要UI
- 入力フォーム
  - 小さな挑戦（必須）
  - その時の感情（必須）
  - 気づき（任意）
  - 次回改善（任意）
- 一覧
  - 日付
  - 挑戦
  - 感情
  - 気づき/改善

### バリデーション
- 挑戦・感情は必須

## S05 メタ認知日記
### 目的
- 目標と実行結果の差分を毎日整理する。

### 主要UI
- 1日1件フォーム
  - 今日の目標（必須）
  - 実際やったこと（必須）
  - 詰まったポイント（任意）
  - 明日やること（必須）
- 保存/更新ボタン
- 過去日記一覧（最新30日）

### 挙動
- 同一日付の再保存は更新扱い

## S06 エクスポート
### 目的
- 期間記録をLLMに貼り付け可能な形式で出力する。

### 主要UI
- 期間選択
  - 過去1日
  - 過去7日
  - 過去30日
- 出力対象チェック
  - タイムライン
  - 習慣記録
  - 運記録
  - メタ認知日記
- 生成ボタン
- 出力テキスト表示欄
- コピー/再生成ボタン

### 出力テンプレート例
```text
以下は私の直近7日間の記録です。傾向分析と次の行動指針を示してください。

[タイムライン]
2026-03-20 09:10 気分4: ...

[習慣開始記録]
読書 2026-03-20 07:30

[運を上げる記録]
挑戦: ...
感情: ...
気づき: ...
次回改善: ...

[メタ認知日記]
目標: ...
実際: ...
詰まり: ...
明日: ...
```

## 4. レスポンシブ方針
- スマホ優先（縦スクロール主体）
- 下部固定タブで主要画面に到達
- 入力フォームは1カラム
- グラフは横スクロール許容
