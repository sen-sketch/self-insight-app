# 要件定義書（MVP）

## 1. 目的
- 個人利用の自己観察Webアプリとして、日々の思考・感情・行動を記録する。
- 1日/1週間/1か月の振り返りに使える形で集計し、LLMへ貼り付け可能なテキストを生成する。
- 初期はAPIを使わず、記録と集計に集中する。

## 2. 対象ユーザー
- 自分自身のみ（マルチユーザー対応は将来対応）。

## 3. スコープ
### 3.1 MVPで実装する機能
- タイムライン投稿（自分用Twitter）
  - 日付・時刻付きの短文投稿
  - 気分スコア（1-5）
  - タグ付け
- 習慣タスク開始記録
  - タスク作成/編集/無効化
  - 開始ボタンで開始時刻を記録
  - 週次で開始時刻の推移表示
- 運を上げる記録
  - 小さな挑戦
  - その時の感情
  - 気づき
  - 次回改善
- メタ認知日記（1日1件）
  - 今日の目標
  - 実際やったこと
  - 詰まったポイント
  - 明日やること
- エクスポート
  - 過去1日/7日/30日の記録を統合
  - LLM貼り付け用テンプレート付き出力
  - コピーボタン

### 3.2 MVP外（将来）
- AI自動分析（API連携）
- 通知（Push/メール）
- マルチデバイス同期最適化
- 画像添付

## 4. 機能要件
### FR-01 タイムライン投稿
- ユーザーは本文・気分スコア・タグを入力し投稿できる。
- 投稿は投稿日時の降順で表示する。

### FR-02 習慣タスク管理
- ユーザーは習慣タスクを作成/編集/有効無効切替できる。
- 有効なタスクは開始記録対象として表示する。
- 習慣タスク一覧では、有効なタスクを無効なタスクより上に表示する。
- 有効なタスクの表示順は、ユーザーが任意に並び替えできる。
- 無効から有効に戻したタスクは、有効なタスク一覧の末尾に表示する。

### FR-03 習慣開始記録
- ユーザーは任意タスクの開始時刻を記録できる。
- 同一タスクに対して1日複数回の記録を許可する。
- 習慣開始記録の一覧取得順序は、開始日時の降順とする。

### FR-04 週次推移表示
- 各習慣タスクごとに、直近7日間の開始時刻を日別に可視化する。
- 同一タスクで1日複数回記録がある場合、当日の最初の開始時刻を集計対象とする。
- 各習慣タスクごとに、7日間の平均開始時刻と前週差を表示する。

### FR-05 運を上げる記録
- ユーザーは挑戦・感情・気づき・次回改善を保存できる。
- 日付で一覧/絞り込みできる。
- 運記録の一覧取得順序は、記録日時の降順とする。

### FR-06 メタ認知日記
- ユーザーは1日1件のメタ認知日記を作成/更新できる。
- 既存日記がある日は更新扱いにする。
- メタ認知日記の一覧取得順序は、日記日付の降順とする。

### FR-07 エクスポート
- ユーザーは期間（1日/7日/30日）を選び統合テキストを生成できる。
- 出力は分析指示文を先頭に含める。
- ボタンでクリップボードにコピーできる。

### FR-08 入力バリデーション
- タイムライン本文、運記録の「小さな挑戦」「その時の感情」、メタ認知日記の「今日の目標」「実際やったこと」「明日やること」は空文字を不可とする。
- 運記録の「気づき」「次回改善」、メタ認知日記の「詰まったポイント」は任意入力とする。
- 気分スコアは 1-5 の整数のみ許可する。
- タグは重複不可とし、空タグは登録不可とする。

### FR-09 編集・削除
- タイムライン投稿、運記録、メタ認知日記は編集および削除できる。
- 削除は論理削除ではなく物理削除とし、MVPでは復元機能を持たない。

## 5. 非機能要件
- レスポンシブ対応（スマホ優先、最小幅360px想定）。
- 主要操作（投稿作成、習慣開始記録、運記録作成、メタ認知日記作成、エクスポートコピー）はダッシュボードから3タップ以内で到達可能。
- 主要画面（ダッシュボード、タイムライン、習慣、運記録、メタ認知、エクスポート）の初回表示は3秒以内（4G相当回線、ミドルレンジスマホ）を目標とする。
- データ保存はローカル端末内（ブラウザストレージ）に限定し、MVPでは外部サーバへ送信しない。

## 6. 前提・制約
- 初期リリースは単一ユーザー利用を前提とする。
- 分析処理はアプリ内で完結せず、LLMへ手動貼り付けで対応する。
- タイムゾーンは `Asia/Tokyo` を標準とする。

## 7. 受け入れ基準（抜粋）
- 4機能（投稿、習慣、運記録、メタ認知）が作成/閲覧できる。
- 各機能の編集/削除が実行できる。
- 7日エクスポート生成で、各カテゴリの記録が時系列で含まれ、分析指示文が先頭に付与される。
- エクスポート結果をコピーボタンでクリップボードへコピーできる。
- 習慣の週次推移はタスクごとに表示され、1日複数回記録時は当日の最初の開始時刻で平均開始時刻が算出される。
- スマホブラウザで入力とコピー操作が問題なく実行できる。

## 8. 実装方針（MVP時点）
### 8.1 使用技術
- フロントエンド: `Next.js 16 (App Router)`
- 言語: `TypeScript`
- スタイリング: `Tailwind CSS`
- データ保存: ブラウザローカル保存（`storage` 層経由で `localStorage` を主利用。将来拡張として `IndexedDB` を利用可能な設計）
- バックエンド: MVPでは外部サーバ送信なし（将来導入時は `Next.js Route Handlers + PostgreSQL + 認証` を想定）

### 8.2 想定ディレクトリと責務
- 実装リポジトリルート: `self-insight-app/`
- `app/`: 画面実装
- `components/`: UI部品
- `lib/`: 集計処理、エクスポート整形、日付処理
- `storage/`: ローカル保存、将来のDB接続切替点
- `docs/`: 要件・画面・DB設計の管理

### 8.3 参照ドキュメント
- 実装方針の詳細・更新は `CLAUDE.md` の「Project Variables」を正本として管理する。

### 8.4 storage 層の一覧取得順序
- `storage` 層の一覧取得は、現在ユーザーのデータのみを返す。
- タイムライン投稿一覧は `postedAt` の降順で返す。
- 習慣タスク一覧は、有効なタスクを先、無効なタスクを後に返し、同一区分内では `sortOrder` の昇順で返す。
- 習慣開始記録一覧は `startedAt` の降順で返す。
- 運記録一覧は `recordedAt` の降順で返す。
- メタ認知日記一覧は `diaryDate` の降順で返す。
